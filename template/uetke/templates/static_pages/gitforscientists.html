Git for scientists

Track changes, work in teams, don't lose data

There are many tools from which scientists can benefit. Git is perhaps the most useful one that gets too often overlooked.

This course is aimed at groups of 10 people who are willing to learn how to achieve a proper workflow and how to collaborate.

Git is useful not only for developing programs but to work in teams. Learn from real-world examples.

Syllabus

Git for scientists will teach you how to exploit a tool that will help you collaborate, track your changes and document your code.

The open source community has quickly shifted to Github to host their projects. The same technology that powers collaborations among big, decentralized teams can be powering also your research organization.

What you are going to learn:

  • Start your own Git project
  • Configure remotes
  • Use different sources: Gitlab, Bitbucket, Github
  • Pull, Push, Merge
  • Work with branches
  • Setup a team project
  • Pull requests
  • Using issues to document changes
  • Better document your code
  • Document Python code with Read The Docs
